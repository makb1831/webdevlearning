HTML + CSS Animation Teaching Demo

This page is made for classroom teaching. It shows the basic ideas of animation in web development: transition, transform, hover effects, zoom effects, fade effects, and simple movement using CSS keyframes.

Start with this rule when teaching: transition controls how change happens, and transform controls what kind of visual change happens.

1. Transition Example

Move the mouse over the button. The color, shadow, and position change smoothly because of transition.

Concept: Smooth change over time
Teaching point: Without transition, the button changes instantly. With transition: 0.4s ease;, the same change becomes animated.

2. Transform Example

Hover on the box to see rotate(), scale(), skew(), and translateX() working together.

Transform
Concept: Shape or position change
Teaching point: transform does not change normal document flow. It changes how the element is visually displayed.

3. Hover Zoom In and Zoom Out

Hover the first box for zoom in and the second box for zoom out. This is a very common UI effect.

Zoom In
Zoom Out
Concept: transform: scale()
Teaching point: Values greater than 1 zoom in, and values smaller than 1 zoom out.

4. Fade Animations

The first box uses an automatic fade animation with @keyframes. The second one fades on hover with opacity.

Automatic Fade In
Hover to Fade Out
Concept: opacity + keyframes
Teaching point: Fade effects are often created by changing opacity from 0 to 1 or from 1 to a lower value.

5. Repeating Movement

This ball moves left to right again and again using CSS only. This is done with @keyframes, not JavaScript.

Concept: Custom animation timeline
Teaching point: Use @keyframes when the animation should run by itself, not only on hover.

6. Teacher Notes

These are the core ideas students should understand after this lesson.

Transition: controls speed, timing, and smoothness.
Transform: controls movement, size, rotation, and skew.
Opacity: controls fade effects.
Keyframes: useful for custom repeated animations.
Some animations are usually JavaScript-dependent, such as: sliders, animations on scroll, counting numbers, drag effects, and animations that react to user clicks in complex ways.
For this file: HTML + CSS only

How to Explain This to Students

  • Use :hover when you want animation only during mouse interaction.
  • Use transition for simple state changes like color, size, position, or opacity.
  • Use transform for rotate, scale, move, and skew effects.
  • Use @keyframes for automatic or repeated animations.
  • Use CSS for simple and medium-level animations before moving to JavaScript.
  • Use JavaScript when animation depends on scroll position, click logic, timers, or dynamic data.
